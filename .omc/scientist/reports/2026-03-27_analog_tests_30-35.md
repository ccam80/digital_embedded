### Circuit

### SPICE Verification
Two independent RC transient simulations — charge phase and discharge phase — verified the timing formula.

Charge phase netlist (cap from Vcc/3 toward Vcc via Ra+Rb):
```spice

.tran 0.01ms 15ms uic
.print tran V(nc)
.end
```
SPICE: cap crosses 2/3*Vcc=3.333V at **t=7.630ms**

Discharge phase netlist (cap from 2Vcc/3 via Rb only):

```spice
.tran 0.01ms 15ms uic
.print tran V(nc)
.end
```

### [FINDING] Frequency = 68.638 Hz (SPICE); formula gives 68.571 Hz; error = 0.097%
[STAT:effect_size] f_formula = 1.44/((Ra+2Rb)·C) = 1.44/(21000·1e−6) = **68.571 Hz**
[STAT:ci] SPICE: t_high=7.630ms, t_low=6.940ms → T=14.569ms → **f=68.638 Hz**; formula error = 0.097%
[STAT:p_value] Formula error 0.097% < 0.1% tolerance — analytical result confirmed
[STAT:n] 2 independent RC transient simulations

| Duty | 52.4% | 52.4% |

### Current test assertion problem
Only checks `outAmps[0] > 0.5` — oscillation amplitude. Does not verify frequency at all.

### [LIMITATION] Full 555 behavioral subcircuit (comparators + SR latch) was not run in ngspice due to self-referential B-source limitations. Frequency verified via two split RC simulations, which is mathematically equivalent to the full charge/discharge cycle.

### Proposed assertion code
```typescript
// SPICE: f=68.638 Hz, T=14.569ms, t_high=7.63ms, t_low=6.94ms, duty=52.4%
// Vcc=5V: output swings 0V to 5V rail-to-rail

await stepAndRead(builder, 1000);
const result = await measurePeaks(builder, 3000);
expect(result).not.toBeNull();

// Amplitude: rail-to-rail swing > 3V (was: > 0.5V — too weak)
const outAmps = [...result!.amplitudes].sort((a, b) => b - a);
expect(outAmps[0]).toBeGreaterThan(3.0);

// Frequency: measure from peak-to-peak spacing
const peaks = result!.peaks;
if (peaks.length >= 3) {
  const periods: number[] = [];
  for (let i = 1; i < peaks.length; i++) {
    periods.push(peaks[i] - peaks[i - 1]);
  }
  const avgPeriodMs = (periods.reduce((a, b) => a + b) / periods.length) * 1000;
  // Allow ±15% for sim timestep quantisation
  expect(avgPeriodMs).toBeGreaterThan(ref.period_ms * 0.85);
  expect(avgPeriodMs).toBeLessThan(ref.period_ms * 1.15);
}
```

## Test 33: SCR Latch (line 1637)

### Circuit
Vcc=12V, R1=1kΩ (anode load), Vtrig=3V, Rg=1kΩ (gate), SW in gate path.

### SPICE Simulations

OFF state — SCR blocking (anode sees full supply):
```spice
Vcc  vcc  0  DC 12
R1   vcc  a  1k
Rscr a    0  100Meg
.op
```
Result: V(a) = **11.988V** (≈ Vcc, tiny leakage through 100MΩ)

ON state — SCR latched (forced 2-diode model for VTM):
```spice
Vcc  vcc  0  DC 12
R1   vcc  a  1k
D1   a    nd1 DMOD
D2   nd1  0   DMOD
.model DMOD D(IS=1e-12 N=1 RS=1)
.op
```
Result: V(a) = **1.217V**, Ia = (12−1.217)/1k = **10.78 mA**

Trigger condition analysis:
- Gate current = (Vtrig − Vgs)/Rg = (3 − 0.7)/1000 = **2.3 mA** >> IGT_min = 0.2 mA → fires
- Ia_hold = 10.78 mA >> IH_min = 5 mA → stays latched

### [FINDING] V(anode) drops from 11.988V (OFF) to 1.217V (ON latched)
[STAT:effect_size] Voltage ratio ON/OFF = 1.217/11.988 = 10.2% — factor 9.8x drop
[STAT:ci] VTM (two diode drops) = 2 × 0.6V = 1.2V; SPICE: 1.217V (1.4% above ideal)
[STAT:p_value] Ia/IH = 10.78/5.0 = 2.16 >> 1 → latching confirmed analytically
[STAT:n] 2 separate DC operating point simulations + analytical holding-current verification

### Current test assertion problem
Only checks `after!.simTime > before!.simTime`. Does not verify V(anode) changed at all.

### [LIMITATION] ngspice built-in SCR element (`.model SCR`) is not supported in this ngspice build. Latching verified via analytical holding-current argument (Ia >> IH) and static ON/OFF DC models. A full transient latch-and-hold simulation would require a charge-storage SCR subcircuit.

### Proposed assertion code
```typescript
// SPICE refs:
//   SCR OFF: V(anode) = 11.988V (~Vcc=12V, all voltage across blocking SCR)
//   SCR ON:  V(anode) = 1.217V  (VTM = 2x Vbe), Ia=10.78mA >> IH=5mA -> latched
const ref = SPICE_REF.scr_latch;

const before = await stepAndRead(builder, 200);
expect(before).not.toBeNull();

// STRONGER: SCR is OFF — anode near Vcc
const voltsBefore = sortedVoltages(before!);
const vAnodeBefore = Math.max(...voltsBefore);
expect(vAnodeBefore).toBeGreaterThan(ref.vcc * 0.85); // > 10.2V

// Close switch to trigger
await clickElementCenter(builder, 'SW');

const after = await stepAndRead(builder, 300);
expect(after).not.toBeNull();
expect(after!.simTime).toBeGreaterThan(before!.simTime);

// STRONGER: SCR has latched — anode at VTM
const voltsAfter = sortedVoltages(after!);
const vAnodeAfter = Math.max(...voltsAfter);
// Latched: V(anode) = VTM ~ 1.2V. Allow 3x for sim model variation.
expect(vAnodeAfter).toBeLessThan(ref.v_anode_on * 3.0); // < ~3.65V
expect(vAnodeAfter).toBeGreaterThan(0.1);                 // not a dead short
```

## [LIMITATION] — Global
1. **Component model defaults unverified**: assertions assume the simulator's LDR/SCR/Triac component defaults match SPICE netlist parameters (VBO, IGT, IH, lux, gamma). Tight assertions require verified defaults.
2. **SCR transient latching**: ngspice built-in SCR unsupported; proven analytically (Ia=10.78mA >> IH=5mA). A subcircuit-based SCR model would confirm the transient.
3. **Triac C1 default**: the test does not set C1. All triac values assume 100nF default; different default changes firing angle and Vrms.
4. **555 full model**: verified via split charge/discharge RC sims. Full behavioral transient was attempted but ngspice B-source self-reference limitations prevent stateful oscillation without subcircuit SR latch.
5. **Integrator SPICE_REF mismatch**: `opamp_integrator.R_ohm=1000` must be corrected to 10000 before proposed assertions reference it.
